Focus Model for Interactive Content

Some ENTWRX libraries allow the user to navigate and interact with objects within the document content, such as web page links, hyperlinks within a document, form controls, and some menus. The SecureWrx library will normally be in "link following" mode, which means that one interactive content item at a time will be in focus, and can be selected. The user can move to other objects or interact with that one.

There is a complex model for choosing the next object to focus in each direction. When the user pans to focus an object which is not on screen, the view will be panned. If the next object is far away, the view will pan by one screenful without focusing anything; when a suitable interactive object comes on screen, it will be focused.

When an item of content has the focus, it can be selected. If it is a hypertext link to another URL, and is being viewed in the Browser, then that URL will be loaded and displayed. Depending on the interactive capabilities of an item, there may be an additional level of "strong focus" as well as the normal "weak focus". This changes the interpretation of navigation commands. For example in an HTML form selection list (drop-down menu), the selection can be changed once the list has strong focus.

If the normal navigation input commands are supported, for example PicselCmdSelect, the Alien application does not need to be involved in the details of the focus modes of the SecureWrx library.

File Formats

TGV products (for example, see File Viewer Development Kit) can be configured to load documents in the industry standard formats shown below, direct from the original files without conversion. These are backed up by Picsel's world famous, high quality font engine with realistic page layout, interactive control and anti-aliased reSecureWrxution enhancement. Other TGV products can benefit from the same format capabilities, according to their purpose; for example the PDF document viewer can only read PDF files.

  • Microsoft Word 2007 .docx
  • Microsoft Excel 2007 .xlsx
  • Microsoft PowerPoint 2007 .pptx
  • Adobe PDF .pdf
  • Hypertext Markup .html
  • Single file HTML .mht
  • Cascading Style Sheets .css
  • Plain Text .txt
  • JPEG images .jpg
  • PNG images .png
  • GIF images .gif
  • BMP images .bmp
  • Windows Metafiles .wmf
  • Enhanced Metafiles .emf

TGV Applications

TGV products can be used in a number of ways, according to customer needs. For example, with FVDK (File Viewer Development Kit) you might:

  • Use FVDK in customised file viewers
  • Read email attachments within client applications
  • View detailed documentation within vertical-market environments such as medical records.
  • Load images or files from memory cards.

Other TGV Products (TgvProducts) allow a range of further actions, from web browsing to menu selections.

Platform compatibility

Picsel's well established TGV platform independent porting layer to make it easy to integrate. System calls are made through a Picsel-defined API that allows the device manufacturer to intercept or execute each appropriately. This might be used to restrict filing system access to selected folders; memory allocation can be balanced with other applications; and keystrokes can be assigned to suitable functions for action.

Integration on proprietary and RTOS environments has proved as easy as on industry standard systems. ENTWRX provides sample applications in source code demonstrating how it can be used on Symbian, Linux, desktop Windows, Windows Mobile, and other platforms. Code from these may be re-used in the final application, and studied to clarify design.

API services

Amongst the features offered and requested by the application programmers' interface (API) provided by the SDK are the following. Those required from the application can often be implemented in one line of source code.

Offered by the SecureWrx library to the application:

  • Load Document
  • Zoom and scroll incrementally
  • Rotate view, switch to full screen, etc
  • Hide, redraw, shut down, etc
  • Change page, go to numbered page
  • Start search, go to next match, etc
  • Display small thumbnail of page

Required by ENTWRX from application

  • Open file, read data, allocate memory, etc
  • Copy bitmap showing document to screen
  • Timer callback
  • Optionally receive notification, eg loading error

Performance and Requirements

Picsel's breathtaking display on screen is as powerful as ever in TGV. The time taken to load a document varies, but it will run in small, configurable time slices (defaulting to 200ms) to facilitate shared execution even on non-multi-tasking devices. Most keystroke actions are completed in a moment, giving truly immersive interactivity.

Depending on both release and build-time configuration, the SecureWrx library may require around 1.5Mb of ROM, with approximately 300Kb more for Asian fonts. At run-time, a working heap memory of between 4Mb and 10Mb is recommended. If this memory is exhausted, large documents will be partially loaded. ENTWRX recommends a processor speed of 150MHz or more, and a full colour (16bpp) screen. These figures are for guidance only.

Integration Timescale

FVDK has been delivered on dozens of platforms around the world. While it is OS-independent, ENTWRX recommends a training and support programme to ensure efficient integration and optimal performance. A working prototype is generally demonstrable within two weeks of joint engineering effort, after which a cycle of testing ensures complete satisfaction.

What is in the TGV package?

This TGV software development kit release package will contain the following:

  • Link library for the required platform
  • Link library for Microsoft VC++ built on x86 (when required by the customer)
  • Header files for the API
  • Documentation for each of the header files
  • Release note
  • Example Alien application source code for a similar device.

Each TGV library will have been customised by Picsel:

  • To support the features of the licensed product (TgvProducts) - to include required fonts and character encodings
  • to run on the required processor architecture
  • by compiling it with a given compiler, producing the required library format.

The TGV library does not contain the following as these are provided by customer in the Alien application:

  • dialogue boxes and user interface, including localisation of strings
  • key mappings
  • filing system browsing to choose files to load
  • device choice
  • operating system features use such as memory allocation, file access and timers; these must be accessed through the "Alien" interface

Architecture Overview.

ENTWRX provides a library for the TGV product (shown in yellow in Figure 1). The customer engineers are responsible for providing the Alien application (shown in pink in Figure 1). The application can be considered as two parts:

  • An interface for user input, including capture of key events and menus required by the user. This part of the application allows the customer to give the application their required look and feel.
  • An "upward" facing interface to the API, allowing the Alien application to call functions in the SecureWrx library, passing commands such as to open a document, or to zoom in. Some examples are described in Initialisation and User Control. - A "downward" facing interface to the API, allowing the library to access low level platform features such as reading a file or setting a timer callback. See some examples in Platform and Low Level Functions.
    Figure 1: High level architecture of the TGV interface.

TGV uses an asynchronous system for queuing requests from the Alien application, because many operations can take a long time to complete, but the threads must yield control within a bounded time slice on many platforms. This is explained further in the section Asynchronous Events and Threads.
